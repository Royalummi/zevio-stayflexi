❌ Error: "Resolved credential object is not valid"

POSSIBLE CAUSES:
================

1. WRONG CREDENTIALS TYPE
   - You might have copied Account ID instead of Access Key ID
   - Or copied something else from wrong section

2. INSUFFICIENT PERMISSIONS
   - Token needs "Object Read & Write" or "Admin Read & Write"
   - Check token permissions in Cloudflare dashboard

3. TOKEN EXPIRED/REVOKED
   - Token might have been deleted
   - Or TTL expired

HOW TO FIX:
===========

Step 1: Verify You Have the RIGHT Credentials
----------------------------------------------
Go to: Cloudflare Dashboard → R2 → Manage R2 API Tokens

You should see your tokens listed. If "zevio-backend-upload" token is there:
1. Click on it
2. Verify permissions (should have Write access)
3. If unsure, DELETE it and create a new one

Step 2: Generate NEW API Token (Recommended)
---------------------------------------------
1. Click "Create API Token" in R2 dashboard
2. Token name: zevio-backend-upload-v2
3. Permissions: Select "Object Read & Write" (minimum)
   OR "Admin Read & Write" (recommended)
4. Choose specific bucket: zevio-images
5. TTL: Forever (or set expiry if needed)
6. Click "Create API token"

Step 3: Copy ALL THREE Values Carefully
----------------------------------------
After creation, you'll see a screen with:

📋 Access Key ID: [long alphanumeric string - 32 chars]
📋 Secret Access Key: [long alphanumeric string - 64 chars]  
📋 Endpoint for S3 clients: https://xxxx.r2.cloudflarestorage.com

⚠️ IMPORTANT: 
- Copy ONE AT A TIME to avoid mixing them up
- Don't close the page until you've saved all three
- These will NOT be shown again!

Step 4: Update .env with New Values
------------------------------------
Replace in backend/.env:

R2_ACCESS_KEY_ID=<paste new Access Key ID here>
R2_SECRET_ACCESS_KEY=<paste new Secret Access Key here>
R2_ENDPOINT=<paste endpoint URL here>

Step 5: Test Again
------------------
cd backend
node test-r2-config.js

TROUBLESHOOTING TIPS:
====================

If still fails:
□ Check no extra spaces before/after values in .env
□ Ensure no quotes around values (should be: KEY=value not KEY="value")
□ Verify bucket name is exactly: zevio-images (case-sensitive)
□ Try creating token with "Admin Read & Write" permissions
□ Check internet connection / firewall not blocking Cloudflare

Need Help?
----------
Share the error message from test-r2-config.js after updating credentials.
